GitHub SSH Authentication
⛔ Error 📌 SSH Key ✅ Resolved
git@github.com: Permission denied (publickey).
fatal: Could not read from remote repository.
No SSH key pair existed on the machine — only a known_hosts file was present. GitHub could not authenticate the connection.
  • Generate an SSH key pair on your machine
  • Start the SSH agent and register the key
  • Copy the public key to your clipboard
  • Add it to GitHub → Settings → SSH and GPG keys → New SSH key
  • Test the connection, then retry the clone
$ ssh-keygen -t ed25519 -C "your@email.com"     # press Enter x3
$ eval "$(ssh-agent -s)"
$ ssh-add ~/.ssh/id_ed25519
$ pbcopy < ~/.ssh/id_ed25519.pub                  # paste into GitHub
$ ssh -T git@github.com                           # verify auth
$ git clone git@github.com:horzuff/robocon2024.git
The response "GitHub does not provide shell access" after authentication is completely normal — it confirms SSH is working.
⛔ Error 📌 Order of Operations ✅ Resolved
/Users/gaorutwe.mosala/.ssh/id_ed25519: No such file or directory
ssh-add was run before ssh-keygen. The key file does not exist until it is explicitly generated.
  • Always run ssh-keygen FIRST — it creates the key file
  • Then run eval "$(ssh-agent -s)" to start the agent
  • Then run ssh-add ~/.ssh/id_ed25519 to register the key
# CORRECT order — generate FIRST, then add
$ ssh-keygen -t ed25519 -C "your@email.com"
$ eval "$(ssh-agent -s)"
$ ssh-add ~/.ssh/id_ed25519

Python Virtual Environment
⛔ Error 📌 Python 2.7 ✅ Resolved
/Library/Frameworks/Python.framework/Versions/2.7/.../Python: No module named venv
The python alias on macOS points to legacy system Python 2.7, which does not include the venv module. Use python3 explicitly.
$ python3 -m venv .venv           # use python3, not python
$ source .venv/bin/activate        # activate the environment
📌 Verification ✅ Resolved
The Oh My Zsh theme can suppress the (.venv) prompt prefix, making it unclear whether activation succeeded. Run these four checks to confirm.
$ which python
→  .../robocon2024/.venv/bin/python  ✅

$ python --version
→  Python 3.13.x  ✅

$ which pip
→  .../robocon2024/.venv/bin/pip  ✅

$ echo $VIRTUAL_ENV
→  /Users/.../robocon2024/.venv  ✅
All four commands returning paths inside .venv confirms the environment is fully isolated and active.

Poetry Dependency Installation
⛔ Error 📌 pyproject.toml ✅ Resolved
The current project's supported Python range (>=3.13) is not compatible with robotframework-pythonlibcore.
robotframework-pythonlibcore requires Python <4,>=3.8 — so it will not be installable for Python >=4.
robotframework-pythonlibcore declares a Python <4 upper bound. Without a matching upper bound in pyproject.toml, Poetry cannot resolve the dependency graph.
  • Open pyproject.toml
  • Add ,<4 to the requires-python field
  • Re-run poetry install
# BEFORE (causes conflict)
requires-python = ">=3.13"

# AFTER (resolves conflict)
requires-python = ">=3.13,<4"   # upper bound prevents Python 4 conflict
⛔ Error 📌 TOML Syntax ✅ Resolved
A missing closing double-quote on the readme value caused TOML to fail to parse the entire file.
# BROKEN — missing closing quote
readme = "README.md

# FIXED
readme = "README.md"   # closing quote was missing
📌 Config ✅ Resolved
package-mode is a Poetry-specific setting. Placing it under the standard [project] table causes an error. It belongs under [tool.poetry].
# WRONG — not a standard [project] key
[project]
package-mode = false

# CORRECT — Poetry-specific section
[tool.poetry]
package-mode = false   # treats project as app, not distributable library

VSCode / RobotCode Extension
⛔ Error 📌 Wrong Interpreter ✅ Resolved
Importing library 'Browser' failed: ModuleNotFoundError: No module named '...'
VSCode was using the system Python interpreter instead of the .venv. Since robotframework-browser is installed inside the venv only, RobotCode could not find it.
  • Press Cmd+Shift+P → type Python: Select Interpreter
  • Choose the .venv entry or paste the path manually
  • Add the interpreter paths to .vscode/settings.json
  • Press Cmd+Shift+P → Developer: Reload Window
// .vscode/settings.json
{
  "python.defaultInterpreterPath": "/Users/gaorutwe.mosala/Documents/
    development/robot-framework/robocon2024/.venv/bin/python",
  "robot.language-server.python":   "/Users/gaorutwe.mosala/Documents/
    development/robot-framework/robocon2024/.venv/bin/python"
}

Robot Framework Test Syntax
⛔ Error 📌 Invalid Format ✅ Resolved
ValueError: Argument 'slowMo' got value '0.00:50' that cannot be converted to timedelta: Invalid timedelta string '0.00:50'
0.00:50 is not a valid timedelta string. The Browser library expects values like 50ms or 1s. Additionally, the Browser. prefix on keywords is redundant when the library is already declared in Settings.
*** Settings ***
Library    Browser

*** Test Cases ***
Test1
    New Browser    browser=chromium    headless=False    slowMo=50ms
    New Context    viewport={"width": 1920, "height": 1080}
    New Page       url=https://www.saucedemo.com
Two fixes applied: slowMo=0.00:50 → slowMo=50ms and Browser.New Browser → New Browser (prefix is redundant).

Platform Differences
📌 Platform ✅ Resolved
The PowerShell command .\\.venv\\Scripts\\Activate.ps1 is Windows-only and will fail on macOS.
# Windows (PowerShell)
.\.venv\Scripts\Activate.ps1

# macOS / Linux (bash / zsh) — use this
$ source .venv/bin/activate
This must be run every time you open a new terminal session. The venv does not persist across sessions automatically.
